# MVP 路线

## 目标

先服务个人自用，再考虑扩展到 AI-heavy 一人公司用户。

MVP 不追求完整公司系统，而是验证一个核心闭环：

> 捕捉输入 -> 形成项目/任务/调研 -> 调用知识和 Playbook -> 产出行动 -> 沉淀记忆。

## Phase 1: 个人驾驶舱

范围：

- 驾驶舱首页
- 待办
- 日历视图
- Inbox
- 项目中心

验证问题：

- 是否能每天打开使用？
- 是否能减少任务和上下文切换成本？
- 是否能明确下一步行动？

## Phase 2: 知识与调研

范围：

- 知识库
- Research 对象
- 资料池
- 调研结论
- 一键转任务、知识、决策

验证问题：

- 是否能把调研从散乱资料变成结构化判断？
- 是否能沉淀为可复用知识？
- 是否能直接推动项目进展？

## Phase 3: Playbook / Skill

范围：

- Playbook 创建与管理
- 输入/步骤/输出模板
- 运行记录
- 与项目、任务、知识、调研关联

验证问题：

- 哪些工作流会反复使用？
- Playbook 是否比临时 prompt 更可靠？
- 是否能逐步升级为可自动执行的 Skill？

## Phase 4: AI Agent 与自动化

范围：

- Agent 任务队列
- 自动日报/周报
- 调研 Agent
- 知识更新提醒
- 审核台

验证问题：

- AI 是否能承担真实工作，而不是只生成文本？
- 用户如何确认、拒绝、修正 AI 结果？
- 自动化日志是否足够可解释？
